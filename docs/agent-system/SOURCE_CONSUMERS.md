# SOURCE_CONSUMERS

Scaffold-only реестр downstream-потребителей Source-снапшота методологии.

**Этот файл в обезличенной upstream-методологии (`agent-system-development`) остаётся scaffold-only: только формат и колонки.** Конкретные потребители (реальные проекты, repository, версии) перечисляются НЕ здесь, а в потребляющем развёртывании — реестр ведёт архитектор того развёртывания. Upstream-методология своих потребителей не перечисляет и не привязывается к конкретным downstream-проектам.

Используется правилом «Source-reminder» (канон — `docs/agent-system/templates/TASK_HEADER_COMMON.md`): если задача меняла методологию/каноны, исполнитель обновляет source-снапшот и в RESULT и блоке «Передача» явно перечисляет зарегистрированных потребителей из этого реестра строкой «Обновить Source-снапшот у зарегистрированных потребителей: …». В обезличенной upstream-методологии реестр пуст, поэтому строка остаётся generic-placeholder.

Ответственная роль за ведение реестра и синхронизацию — `source-steward` (канон ролей — `docs/agent-system/ROLE_MODEL.md`).

## Принципы

- Source of truth — GitHub-файлы methodology repository. Снапшоты у потребителей являются derived context (политика снапшота — `docs/agent-system/source/README.md`).
- Реестр перечисляет, **кому** нужен снапшот; синхронизация выполняется в каждом потребителе по его branch-модели через PR, а не прямой правкой.
- Реестр не содержит private data, credentials, tokens и внутренних кодовых имён; только публично известные repository-указатели.
- Записи добавляются только в потребляющем развёртывании; в обезличенном шаблоне реальных проектов нет.

## Формат реестра (заполняется в потребляющем развёртывании)

| project | repo | source-path | adopted-version | last-synced | notes |
|---------|------|-------------|-----------------|-------------|-------|
| `<напр. my-project>` | `<url>` | `<путь к Source в проекте>` | `<версия/тег методологии>` | `<дата/SHA>` | `<...>` |

## Как обновлять реестр (в потребляющем развёртывании)

- Новый потребитель — добавить строку с repository-указателем, путём к Source, принятой версией методологии и датой/SHA последней синхронизации.
- Потребитель больше не зависит от снапшота — пометить строку как архивную в `notes` (не удалять историю без отдельного решения пользователя).
